## 🔬 Proxy定義の妥当性

### Feature nodes相当 (8個)
**物理的意味**:
各特徴量は観測可能な物理量であり、BLSの入力層に相当
- 関節角度変化: delta_theta
- 足先変位: end_disp
- 経路長: path_length
- 経路の直進性: path_straightness
- 反転回数: reversals
- 本体高さ: base_height
- 姿勢角: max_roll, max_pitch

### Enhancement nodes相当 (24個)
**計算ステップ**:
特徴量から診断結果を導く中間処理に相当
1. **ロバスト統計**: 6試行の中央値計算 → 6ステップ
2. **正規化**: 各特徴量の0-1変換 → 6ステップ
3. **判定ロジック**: 
   - BURIED判定: 2条件チェック
   - TRAPPED判定: 4条件チェック（優先判定含む）
   - TANGLED判定: 2条件チェック
   - スコア計算: 4ステップ
   - 合計: 12ステップ

### BLSとの対応関係
```
BLS:
  入力 → [特徴抽出] → [非線形変換・拡張] → 出力
         Feature nodes   Enhancement nodes

本手法:
  観測 → [特徴計算8個] → [中央値・判定24ステップ] → 診断
         ≈Feature nodes   ≈Enhancement nodes
```
